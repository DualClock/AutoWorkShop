# 🔧 Исправление ошибки "Invalid column name 'PasswordHash'"

## Проблема
При запуске приложения через `.bat` файл возникает ошибка:
```
Invalid column name 'PasswordHash'
```

## Причина
В вашей базе данных на сервере `localhost` используется **старая версия таблицы Users** с полем `PasswordHash`, в то время как текущая версия приложения использует поле `Password`.

Это происходит потому, что:
- Visual Studio использует LocalDB `(localdb)\MSSQLLocalDB`
- При запуске через `.bat` используется полный SQL Server `localhost`
- В полном SQL Server осталась старая версия базы данных

## Решение

### Способ 1: Выполнить скрипт исправления (РЕКОМЕНДУЕТСЯ)

1. Откройте **SQL Server Management Studio**
2. Подключитесь к серверу `localhost`
3. Откройте файл `04_FixPasswordHash.sql`
4. Выполните скрипт (F5 или кнопка Execute)

Скрипт автоматически:
- Проверит наличие старого поля `PasswordHash`
- Создаст новое поле `Password`
- Скопирует данные из старого поля в новое
- Удалит старое поле `PasswordHash`

### Способ 2: Пересоздать базу данных

Если вы хотите начать с чистой базы данных:

1. Откройте **SQL Server Management Studio**
2. Подключитесь к серверу `localhost`
3. Выполните скрипт `03_DatabaseReset.sql` (очистка всех данных)
4. Затем выполните скрипт `01_CreateDatabase_Full.sql` (создание и заполнение)

**ВНИМАНИЕ:** Это удалит все данные из базы данных!

## Проверка

После исправления выполните проверку:

```sql
USE AutoWorkshopDB;
GO

-- Проверка структуры таблицы Users
SELECT COLUMN_NAME, DATA_TYPE, IS_NULLABLE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME = 'Users'
ORDER BY ORDINAL_POSITION;
GO

-- Проверка пользователей
SELECT Id, Login, Role, IsActive FROM Users;
GO
```

Вы должны увидеть поле `Password` (не `PasswordHash`!).
