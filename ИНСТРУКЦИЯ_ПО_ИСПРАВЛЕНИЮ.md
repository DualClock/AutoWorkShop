# 🔧 Исправление ошибки "Invalid column name 'PasswordHash'"

## Проблема
При запуске опубликованного приложения (`Publish/AutoWorkShop.exe`) возникает ошибка:
```
Invalid column name 'PasswordHash'
```

При этом при запуске из Visual Studio ошибки нет.

## Причина
**АКТУАЛИЗИРОВАНО В ВЕРСИИ 1.0.3:**

Проблема была в том, что опубликованное приложение использовало строку подключения из `appsettings.json`:
```
Server=localhost;Database=AutoWorkshopDB;...
```

А Visual Studio использовал строку из `AppDbContext.cs`:
```
Data Source=(localdb)\MSSQLLocalDB;Database=AutoWorkshopDB;...
```

Это означало, что:
- VS подключался к LocalDB (встроенная версия SQL Server)
- Опубликованное приложение подключалось к `localhost` (полный SQL Server)
- В базе данных на `localhost` было старое поле `PasswordHash`

**В версии 1.0.3:**
- ✅ `AppDbContext.cs` теперь читает строку подключения из `appsettings.json`
- ✅ По умолчанию используется `(localdb)\MSSQLLocalDB` (как в VS)
- ✅ Строки подключения унифицированы

## Решение

### Способ 1: Обновиться до версии 1.0.3 (РЕКОМЕНДУЕТСЯ)

Просто переопубликуйте проект:
```bash
cd c:\Users\Nikita\source\repos\AutoWorkShop
dotnet publish --configuration Release -r win-x64 --self-contained false -o Publish
```

Теперь приложение будет использовать ту же базу данных, что и Visual Studio.

### Способ 2: Выполнить скрипт исправления

Если вы хотите использовать полный SQL Server (`localhost`):

1. Откройте **SQL Server Management Studio**
2. Подключитесь к серверу `localhost`
3. Откройте файл `04_FixPasswordHash.sql`
4. Выполните скрипт (F5 или кнопка Execute)

Скрипт автоматически:
- Проверит наличие старого поля `PasswordHash`
- Создаст новое поле `Password`
- Скопирует данные из старого поля в новое
- Удалит старое поле `PasswordHash`

### Способ 2: Пересоздать базу данных

Если вы хотите начать с чистой базы данных:

1. Откройте **SQL Server Management Studio**
2. Подключитесь к серверу `localhost`
3. Выполните скрипт `03_DatabaseReset.sql` (очистка всех данных)
4. Затем выполните скрипт `01_CreateDatabase_Full.sql` (создание и заполнение)

**ВНИМАНИЕ:** Это удалит все данные из базы данных!

### Способ 3: Обновить базу данных через EF Core

Если у вас установлен .NET SDK:

```bash
cd c:\Users\Nikita\source\repos\AutoWorkShop\AutoWorkShop
dotnet ef database update
```

## Проверка

После исправления выполните проверку:

```sql
USE AutoWorkshopDB;
GO

-- Проверка структуры таблицы Users
SELECT COLUMN_NAME, DATA_TYPE, IS_NULLABLE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME = 'Users'
ORDER BY ORDINAL_POSITION;
GO

-- Проверка пользователей
SELECT Id, Login, Role, IsActive FROM Users;
GO
```

Вы должны увидеть поле `Password` (не `PasswordHash`!).
