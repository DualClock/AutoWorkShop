## 2. УСТАНОВКА SQL SERVER

### Вариант А: SQL Server Express (рекомендуется для работы)

1. **Скачайте SQL Server Express**
   - Перейдите на страницу: https://www.microsoft.com/sql-server/sql-server-downloads
   - Нажмите "Скачать бесплатно" для SQL Server Express

2. **Запустите установщик**
   - Выберите тип установки **"Базовая"**
   - Примите условия лицензии
   - Дождитесь завершения установки

3. **Запишите имя экземпляра**
   - Обычно: `localhost\SQLEXPRESS` или `.\SQLEXPRESS`
   - Также будет предложено установить SSMS (Management Studio)

### Вариант Б: SQL Server Management Studio (SSMS)

1. **Скачайте SSMS**
   - Перейдите на страницу: https://learn.microsoft.com/sql/ssms/download-ssms
   - Скачайте и установите

2. **Запустите SSMS**
   - В поле "Имя сервера" введите: `localhost` или `localhost\SQLEXPRESS`
   - Нажмите "Подключить"

## 4. УСТАНОВКА ПРИЛОЖЕНИЯ

### Шаг 4.1: Копирование файлов

1. Скопируйте папку с приложением в нужное место, например:
   ```
   C:\Program Files\AutoWorkShop\
   ```
   или
   ```
   C:\AutoWorkShop\
   ```

2. Убедитесь, что в папке присутствуют файлы:
   - `AutoWorkShop.exe`
   - `appsettings.json`
   - `AutoWorkShop.dll`
   - и другие файлы

### Шаг 4.2: Настройка подключения к БД

1. Откройте файл **`appsettings.json`** в текстовом редакторе

2. Измените строку подключения в зависимости от вашей конфигурации:

   **Для SQL Server Express:**
   ```json
   {
       "ConnectionStrings": {
           "DefaultConnection": "Server=localhost\\SQLEXPRESS;Database=AutoWorkshopDB;Trusted_Connection=True;TrustServerCertificate=True;"
       }
   }
   ```

   **Для локального SQL Server:**
   ```json
   {
       "ConnectionStrings": {
           "DefaultConnection": "Server=localhost;Database=AutoWorkshopDB;Trusted_Connection=True;TrustServerCertificate=True;"
       }
   }
   ```

   **Для LocalDB:**
   ```json
   {
       "ConnectionStrings": {
           "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=AutoWorkshopDB;Integrated Security=True;TrustServerCertificate=True;"
       }
   }
   ```

3. Сохраните файл

### Шаг 4.3: Создание ярлыка

1. Щёлкните правой кнопкой мыши на `AutoWorkShop.exe`
2. Выберите **"Создать ярлык"**
3. Переместите ярлык на рабочий стол

## 6. НАСТРОЙКА ПОЛЬЗОВАТЕЛЕЙ

### Шаг 6.1: Добавление нового пользователя

1. Войдите под администратором
2. Перейдите на вкладку **"👥 Пользователи"**
3. Нажмите кнопку **"➕ Добавить"**
4. Заполните поля:
   - **Логин** — фамилия или имя сотрудника
   - **Пароль** — временный пароль (6+ символов)
   - **Роль** — выберите из списка
   - **Сотрудник** — привяжите к записи в справочнике сотрудников
5. Нажмите **"Сохранить"**

### Шаг 6.2: Описание ролей

| Роль | Права доступа |
|------|---------------|
| **Администратор** | Полный доступ ко всем функциям, управление пользователями |
| **Мастер** | Создание и редактирование заказов, работа с клиентами |
| **Механик** | Просмотр заказов, отметка о выполнении работ |
| **Бухгалтер** | Работа с финансами, квитанциями, отчётами |
| **Кладовщик** | Учёт запчастей, складские операции |

### Шаг 6.3: Настройка прав доступа

Подробное описание прав доступа см. в файле `РОЛИ_ИНСТРУКЦИЯ.md`

## 7. ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Ошибка: "Не удалось подключиться к базе данных"

**Причины:**
- SQL Server не запущен
- Неверное имя сервера в настройках
- База данных не создана

**Решение:**
1. Проверьте, запущена ли служба SQL Server
   - Нажмите `Win+R`, введите `services.msc`
   - Найдите "SQL Server (MSSQLSERVER)" или "SQL Server (SQLEXPRESS)"
   - Убедитесь, что статус "Выполняется"

2. Проверьте файл `appsettings.json`
   - Убедитесь в правильности имени сервера
   - Проверьте название базы данных

3. Убедитесь, что база данных существует
   - Откройте SSMS
   - Проверьте наличие `AutoWorkshopDB`

### Ошибка: "Неверный логин или пароль"

**Причины:**
- Неправильная раскладка клавиатуры
- Включён CapsLock
- База данных не заполнена

**Решение:**
1. Проверьте раскладку клавиатуры (RU/EN)
2. Отключите CapsLock
3. Убедитесь, что скрипт `01_CreateDatabase_Full.sql` выполнен успешно
4. Проверьте таблицу `Users` через SSMS

### Кнопки не отображаются в интерфейсе

**Причины:**
- У пользователя недостаточно прав
- Проблема с кодировкой

**Решение:**
1. Войдите под администратором
2. Проверьте роль пользователя
3. При необходимости выполните скрипт `fix_encoding.sql`

### Приложение не запускается

**Причины:**
- Не установлен .NET 10.0 Runtime
- Повреждены файлы приложения

**Решение:**
1. Установите .NET 10.0 Runtime
   - Скачайте с официального сайта Microsoft
2. Переустановите приложение
