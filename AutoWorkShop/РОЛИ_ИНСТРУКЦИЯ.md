# Инструкция по системе ролей в АвтоМаст ИС

## 📋 Обзор ролей

В системе реализованы 3 уровня доступа:

### 1. **Администратор (Admin)** 🔴
Полный доступ ко всем функциям системы:
- ✅ Просмотр всех данных
- ✅ Добавление/изменение/удаление любых данных
- ✅ Управление пользователями (создание, редактирование, удаление, блокировка)
- ✅ Доступ ко всем вкладкам и функциям

### 2. **Менеджер (Manager)** 🟠
Расширенный доступ без управления пользователями:
- ✅ Просмотр всех данных
- ✅ Добавление/изменение данных
- ❌ Удаление данных (только администратор)
- ❌ Управление пользователями

### 3. **Рабочий (Worker)** 🟢
Базовый доступ для работы:
- ✅ Просмотр данных
- ✅ Создание запросов и заказов
- ❌ Добавление/изменение справочной информации
- ❌ Удаление данных
- ❌ Управление пользователями


### Шаг 2: Создание нового пользователя (Рабочий)

1. Войдите под администратором (login: `admin`, password: `admin123`)
2. Перейдите на вкладку **"👥 Пользователи"**
3. Нажмите кнопку **"➕ Добавить"**
4. Заполните данные:
   - **Логин**: например, `ivanov`
   - **Пароль**: например, `password123`
   - **Роль**: выберите "Рабочий"
   - **Сотрудник**: выберите сотрудника из списка (или "Не назначен")
5. Нажмите **"Сохранить"**

### Шаг 3: Проверка прав доступа

1. Выйдите из системы (кнопка "Выход")
2. Войдите под новым пользователем
3. Проверьте, что:
   - Кнопки "Добобавить", "Изменить", "Удалить" скрыты в разделе "Отделы"
   - Вкладка "Пользователи" скрыта или кнопки неактивны
   - Можно создавать заказы и запросы

## 📝 Примеры использования

### Пример 1: Создание учётной записи для нового сотрудника

**Задача:** Дать доступ к системе новому менеджеру Петрову.

**Решение:**
1. Администратор заходит в "Пользователи"
2. Нажимает "Добавить"
3. Вводит:
   - Логин: `petrov`
   - Пароль: `Manager2026!`
   - Роль: `Менеджер`
   - Сотрудник: `Петров П.П.` (выбрать из списка)
4. Сохраняет
5. Сообщает логинов/пароль сотруднику

### Пример 2: Ограничение прав для временного сотрудника

**Задача:** Дать доступ стажёру только для просмотра и создания запросов.

**Решение:**
1. Создать пользователя с ролью "Рабочий"
2. При необходимости заблокировать после окончания стажировки:
   - Выбрать пользователя в списке
   - Нажать "🚫 Заблокировать"

### Пример 3: Повышение сотрудника

**Задача:** Изменить роль с "Рабочий" на "Менеджер".

**Решение:**
1. Администратор выбирает пользователя
2. Нажимает "✏️ Изменить"
3. Меняет роль на "Менеджер"
4. Сохраняет

## 🔒 Безопасность

### Рекомендации по паролям:
- Минимум 8 символов
- Используйте буквы и цифры
- Не используйте одинаковые пароли для всех

### Блокировка пользователей:
- При увольнении сотрудника сразу блокируйте учётную запись
- Не удаляйте пользователей, если с ними связаны исторические данные
- Используйте блокировку вместо удаления

## ❓ Частые вопросы

**Q: Как восстановить пароль пользователю?**  
A: Администратор может изменить пароль в диалоге редактирования пользователя.

**Q: Можно ли удалить администратора?**  
A: Нет, система не позволит удалить самого себя.

**Q: Что делать, если все администраторы заблокированы?**  
A: Выполните SQL-запрос в базе данных:
```sql
UPDATE Users SET IsActive = 1, Role = 'Admin' WHERE Login = 'admin';
```

**Q: Как увидеть, когда пользователь последний раз входил?**  
A: В таблице пользователей есть колонка "Последний вход".
