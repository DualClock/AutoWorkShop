## ТРЕБОВАНИЯ К СИСТЕМЕ
================================================================================

### Операционная система:
- Windows 10/11 (64-bit)
- Windows Server 2016 или новее

### Необходимое ПО:
1. **.NET 10.0 Runtime** (скачать: https://dotnet.microsoft.com/download)
2. **SQL Server 2016+** или **SQL Server Express** (бесплатно)
   - Альтернатива: LocalDB (входит в состав Visual Studio)

### Минимальные характеристики:
- Процессор: 2 ядра, 2.0 ГГц
- ОЗУ: 4 ГБ
- Свободное место: 500 МБ

================================================================================
## ЭТАП 1: УСТАНОВКА SQL SERVER
================================================================================

### Вариант А: SQL Server Express (рекомендуется)

1. Скачайте SQL Server Express:
   https://www.microsoft.com/sql-server/sql-server-downloads

2. Запустите установщик, выберите "Базовая" установка

3. После установки запишите имя экземпляра (обычно: localhost\SQLEXPRESS)

### Вариант Б: LocalDB (для тестирования)

1. Установите Visual Studio с компонентом "LocalDB"
2. Имя экземпляра: (localdb)\MSSQLLocalDB

================================================================================
## ЭТАП 2: СОЗДАНИЕ БАЗЫ ДАННЫХ
================================================================================

1. Запустите **SQL Server Management Studio (SSMS)**

2. Подключитесь к вашему экземпляру SQL Server

3. Откройте файл: `Deploy\01_CreateDatabase.sql`

4. Нажмите **F5** (Выполнить)

5. Дождитесь сообщения "БАЗА ДАННЫХ УСПЕШНО СОЗДАНА!"

6. Для проверки откройте и выполните:
   `Deploy\03_DatabaseCheck.sql`

================================================================================
## ЭТАП 3: НАСТРОЙКА ПРИЛОЖЕНИЯ
================================================================================

1. Скопируйте папку с приложением в нужное место, например:
   `C:\Program Files\AutoWorkShop\`

2. Откройте файл `appsettings.json` в текстовом редакторе

3. Измените строку подключения при необходимости:

   Для SQL Server Express:
   ```json
   "DefaultConnection": "Server=localhost\\SQLEXPRESS;Database=AutoWorkshopDB;Trusted_Connection=True;TrustServerCertificate=True;"
   ```

   Для LocalDB:
   ```json
   "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=AutoWorkshopDB;Integrated Security=True;TrustServerCertificate=True;"
   ```

4. Сохраните файл

================================================================================
## ЭТАП 4: ЗАПУСК ПРИЛОЖЕНИЯ
================================================================================

1. Запустите файл `AutoWorkShop.exe`

2. Выполните вход под учётной записью администратора:
   - Логин: **admin**
   - Пароль: **admin123**

3. Смените пароль администратора после первого входа!

================================================================================
## ЭТАП 5: СОЗДАНИЕ ПОЛЬЗОВАТЕЛЕЙ
================================================================================

1. Войдите под администратором

2. Перейдите на вкладку **"👥 Пользователи"**

3. Нажмите **"➕ Добавить"**

4. Заполните данные:
   - Логин: фамилия сотрудника
   - Пароль: временный пароль
   - Роль: выберите из списка
     - **Администратор** — полный доступ
     - **Менеджер** — добавление/изменение без удаления
     - **Рабочий** — только просмотр и создание запросов

5. Нажмите **"Сохранить"**

================================================================================
## УДАЛЕНИЕ ПРОГРАММЫ
================================================================================

1. Закройте приложение

2. Удалите папку с приложением

3. В SQL Server выполните:
   ```sql
   USE master;
   GO
   DROP DATABASE AutoWorkshopDB;
   GO
   ```

## ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
================================================================================

### Ошибка: "Нет подключения к базе данных"

**Решение:**
1. Проверьте, запущена ли служба SQL Server
2. Проверьте правильность строки подключения в appsettings.json
3. Убедитесь, что база данных AutoWorkshopDB существует

### Ошибка: "Неверный логин или пароль"

**Решение:**
1. Убедитесь, что скрипт 01_CreateDatabase.sql выполнен успешно
2. Проверьте раскладку клавиатуры и CapsLock

### Кнопки не отображаются в интерфейсе

**Решение:**
1. Выполните скрипт `Deploy\02_UpdateDatabase.sql`
2. Перезапустите приложение

================================================================================

При возникновении проблем:

1. Проверьте логи приложения (папка Logs, если существует)

2. Выполните скрипт проверки: `Deploy\03_DatabaseCheck.sql`

3. Убедитесь, что выполнены все требования к системе
